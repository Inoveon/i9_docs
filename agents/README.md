# Agents e Skills - i9ON/Inoveon

Catálogo de agents e skills utilizados no desenvolvimento.

## 📋 Categorias

### Scrum (6 agents)
- **scrum-po** - Product Owner - representa stakeholders, gerencia Product Backlog
- **scrum-sm** - Scrum Master - facilita processo Scrum, remove impedimentos
- **scrum-daily** - Daily Standup - conduz reunião diária de 15 minutos
- **scrum-planning** - Sprint Planning - conduz planejamento da sprint
- **scrum-review** - Sprint Review - conduz revisão da sprint
- **scrum-retro** - Retrospectiva - facilita retrospectiva do time

### Desenvolvimento (6 agents)
- **dev-frontend** - Next.js/React/TypeScript/TailwindCSS
- **dev-backend** - Node.js/Express/Prisma/TypeScript/PostgreSQL
- **dev-mobile** - Flutter/Dart com Riverpod
- **dev-database** - Prisma 7/PostgreSQL com seeds modulares
- **dev-devops** - CI/CD GitHub Actions, Docker
- **dev-architect** - Arquitetura com ADRs e diagramas

### Quality Assurance (6 agents)
- **qa-tester** - Cria e executa testes, valida funcionalidades
- **qa-validator** - Analisa código e cria issues detalhadas
- **qa-ui-auditor** - Analisa violações de padrões de UI/UX
- **qa-ui-fixer** - Aplica correções de UI/UX
- **qa-acentuacao** - Corretor de acentuação PT-BR
- **qa-homologation** - Gera resumos de homologação

### Padrões (2 agents)
- **api-corporativo** - Padrão corporativo de API REST com RFC 9457
- **crud-frontend** - Padrões de CRUD Next.js/React

### Validação (1 agent)
- **val-perf** - Valida performance PostgreSQL, índices, queries

### Business (1 agent)
- **orcamento** - Especialista em Orçamentos Protheus - Grupo Aldo

## 🚀 Como Usar

Cada agent possui documentação detalhada com:
- **Objetivo** - O que o agent faz
- **Contexto** - Quando usar
- **Comandos** - Como invocar
- **Exemplos** - Casos de uso
- **Configuração** - Parâmetros e setup
